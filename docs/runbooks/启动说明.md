# 医院知识库管理平台启动说明

本文档说明如何在本地启动完整项目：MongoDB、Qdrant、Go 后端、React 前端。

## 1. 环境要求

- Node.js 20 或更高版本
- npm 10 或更高版本
- Go 1.22 或更高版本
- Docker 和 Docker Compose

当前代码仓库中后端使用 Go + chi，前端使用 JavaScript + React + React Router。

Docker 安装和数据库查看请阅读：

```text
docs/runbooks/docker和数据库查看.md
```

## 2. 启动数据库和向量库

先确认 Docker 已启动：

```bash
docker --version
docker compose version
```

在项目根目录执行：

```bash
docker compose up -d mongodb qdrant
```

查看容器状态：

```bash
docker compose ps
```

MongoDB 会自动执行：

```text
backend/scripts/mongo-init.js
```

该脚本会创建集合和索引，相当于本项目 MongoDB 的建表脚本。

Qdrant 向量集合可手动初始化：

```bash
cd backend
sh scripts/qdrant-init.sh
```

后端启动时也会尝试自动创建 Qdrant collection。

## 3. 配置后端环境变量

复制环境模板：

```bash
cd backend
cp .env.example .env
```

编辑 `.env`，填写以下关键配置：

```text
DEEPSEEK_API_KEY=你的DeepSeek密钥
QWEN_EMBEDDING_BASE_URL=https://ark.cn-beijing.volces.com/api/v3
QWEN_EMBEDDING_API_KEY=你的火山引擎方舟密钥
QWEN_EMBEDDING_MODEL=doubao-embedding-vision-251215
QWEN_EMBEDDING_DIMENSION=向量维度
```

说明：

- DeepSeek 只负责对话生成。
- 火山引擎方舟豆包向量模型负责文档向量和查询向量生成。
- 如果暂时不填写模型密钥，后端会使用本地演示逻辑生成流式回复和确定性向量，便于先跑通界面。

## 4. 启动后端

```bash
cd backend
go mod download
set -a
. ./.env
set +a
go run ./cmd/server
```

后端默认地址：

```text
http://localhost:8080
```

健康检查：

```bash
curl http://localhost:8080/health
```

默认登录账号：

```text
账号：admin
密码：admin123
```

## 5. 启动前端

```bash
cd frontend
cp .env.example .env
npm install
npm run dev
```

前端默认地址：

```text
http://localhost:5173
```

## 6. 构建命令

前端构建：

```bash
cd frontend
npm run build
```

后端构建：

```bash
cd backend
go build ./cmd/server
```

## 7. 功能验证路径

1. 打开 `http://localhost:5173`
2. 使用 `admin / admin123` 登录
3. 进入“知识库管理”，查看内置知识库
4. 选择知识库上传 PDF、Word、Excel、Markdown 或文本文件
5. 进入“对话管理”，新建对话
6. 输入问题，确认回答逐字流式输出
7. 点击回复详情，查看引用来源、耗时和提示词上下文

## 8. 项目结构说明

前端核心目录：

```text
frontend/src/api/        接口客户端和 SSE 解析
frontend/src/contexts/   登录态和用户上下文
frontend/src/layouts/    登录后统一布局
frontend/src/pages/      登录、对话、知识库页面
frontend/src/features/   对话和知识库业务组件、hooks
```

后端核心目录：

```text
backend/internal/httpapi/ chi 路由、中间件、请求 DTO、handler、响应工具
backend/internal/store/   MongoDB 数据访问，按 auth/chat/knowledge/ingestion/rag/audit 拆分
backend/internal/rag/     RAG 检索和提示词上下文组装
backend/internal/ingestion/ 文档入库 worker
```

## 9. 常见问题

### 后端提示 MongoDB 连接失败

确认 Docker 容器已启动：

```bash
docker compose ps
```

进入容器查看 MongoDB 内容：

```bash
docker exec -it medical-agent-mongodb mongosh
use medical_agent
show collections
db.knowledge_bases.find().pretty()
```

### 向量模型维度不匹配

确认 `QWEN_EMBEDDING_DIMENSION` 与实际模型返回向量维度一致，并重新创建 Qdrant collection。

### DeepSeek 没有配置

后端仍可用本地演示回答跑通流程。正式使用前必须填写 `DEEPSEEK_API_KEY`。
