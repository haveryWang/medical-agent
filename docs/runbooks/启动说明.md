# 医院行政智策平台启动说明

启动脚本统一放在项目根目录的 `scripts/` 文件夹。

## 一键启动

首次启动建议先准备依赖和 `.env` 文件：

```bash
sh scripts/setup.sh
```

然后启动完整本地环境：

```bash
sh scripts/start-all.sh
```

启动后访问：

```text
前端：http://localhost:5173
后端：http://localhost:8080
```

默认账号：

```text
账号：admin
密码：admin123
```

## 常用脚本

```bash
sh scripts/setup.sh          # 创建 .env，安装前端依赖，下载后端依赖
sh scripts/start-db.sh       # 启动 MongoDB / Qdrant，并初始化 Qdrant collection
sh scripts/start-backend.sh  # 启动 Go 后端
sh scripts/start-frontend.sh # 启动 React 前端
sh scripts/start-all.sh      # 启动数据库、后端、前端
sh scripts/build.sh          # 构建后端和前端
sh scripts/test.sh           # 运行后端测试和前端构建检查
sh scripts/reset-database.sh # 手动清空业务数据，保留/重建 admin 账号
```

`start-*` 启动脚本不会清空知识库、文档、会话、复盘记录、政策文件库或向量数据。只有手动执行 `scripts/reset-database.sh` 才会重置 MongoDB 业务集合和 Qdrant collection，并保留/重建带管理权限的 admin 账号。

如果只是模型向量维度变更导致 Qdrant collection 不匹配，执行：

```bash
sh scripts/reset-vector-index.sh
```

该脚本只清理 chunks、重建 Qdrant collection，并重新排队已有文档入库；知识库、原始文件、用户和会话不会被删除。

## 模型配置

`scripts/setup.sh` 会自动创建：

```text
backend/.env
frontend/.env
```

正式使用前，建议在 `backend/.env` 里填写：

```text
VOLCENGINE_API_KEY=你的火山引擎方舟密钥
```

也可以登录页面后点击顶栏“系统设置”，在弹窗里保存模型配置到数据库。当前默认模型：

```text
对话模型：deepseek-v4-flash-260425
向量模型：doubao-embedding-vision-251215
Base URL：https://ark.cn-beijing.volces.com/api/v3
向量维度：2048
```

## 单独启动

如果想分开看日志，可以开三个终端：

```bash
sh scripts/start-db.sh
sh scripts/start-backend.sh
sh scripts/start-frontend.sh
```
