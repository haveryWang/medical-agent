# OpenSpec Workflow

Use OpenSpec when a change affects requirements, behavior, public interfaces, or
architecture.

## Suggested Flow

1. Explore the request and existing context.
2. Create or update an OpenSpec change under `openspec/changes/<change-id>/`.
3. Capture requirements in delta specs.
4. Implement only after the intended change is clear.
5. Sync or archive specs after implementation is accepted.

## Project Skills

Project-local OpenSpec skills are available under `.codex/skills/`:

- `openspec-explore`
- `openspec-propose`
- `openspec-apply-change`
- `openspec-sync-specs`
- `openspec-archive-change`
